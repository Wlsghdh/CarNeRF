# CarNeRF

> **AI가 분석하고, 3D로 보여주는 투명한 중고차 거래 플랫폼**

---

## 목차

**1. 문제인식** — 중고차 시장의 구조적 문제와 우리가 본 기회
**2. 실현가능성** — 이걸 어떻게 만들었고, 왜 우리만 할 수 있는가
**3. 성장전략** — 돈은 어떻게 벌고, 앞으로 어떻게 키울 것인가
**4. 팀 구성** — 누가 무엇을 맡고 있는가

---

# 1. 문제인식

## 중고차, 지금 어떻게 사고 파나요?

지금 중고차를 사려면 크게 두 가지 선택지가 있습니다.

**첫 번째, 엔카나 KB차차차 같은 딜러 기반 플랫폼.**
여기는 차량 정보가 체계적이긴 하지만, 결국 딜러를 통해 거래가 이루어집니다. 딜러 수수료가 붙고, 엔카의 경우 매물 등록비만 해도 돈이 들어갑니다. 판매자 입장에서는 비용이 부담되고, 구매자 입장에서는 그 비용이 차 값에 반영되어 결국 더 비싸게 사게 됩니다. 게다가 성능기록부니 진단서니 전문 용어로 가득 차 있어서, 차를 잘 모르는 일반인은 읽어봐도 뭐가 뭔지 모릅니다.

**두 번째, 당근마켓 같은 직거래 플랫폼.**
딜러 없이 개인 간 거래니까 수수료는 없습니다. 그런데 문제는 **불안하다**는 겁니다. 사진 몇 장이 전부고, 차량 상태를 검증할 방법이 없습니다. 판매자가 "사고 없음"이라고 써놓으면 믿을 수밖에 없고, 가격이 적정한 건지 비싼 건지 판단할 근거도 없습니다. 결국 "이 가격이 맞는 건가?"라는 불신 속에서 거래가 이루어집니다.

```
┌─────────────────────────────────┐
│  딜러 기반 (엔카, KB차차차)       │
│  ✓ 정보는 있음                   │
│  ✗ 딜러 수수료, 등록비 → 비쌈     │
│  ✗ 전문 용어 → 일반인 이해 불가   │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│  직거래 (당근마켓)               │
│  ✓ 수수료 없음, 저렴             │
│  ✗ 차량 상태 검증 수단 없음       │
│  ✗ 가격 적정성 판단 불가 → 불신   │
└─────────────────────────────────┘
```

이 두 시장의 단점은 서로 정반대입니다. 딜러 플랫폼은 "비싸다", 직거래 플랫폼은 "불안하다".

저희는 여기서 기회를 봤습니다. **이 두 가지 문제를 동시에 해결하면, 양쪽 시장의 사용자를 모두 끌어올 수 있는 독점적 포지션을 확보할 수 있다**고 판단했습니다.

---
        
## 그래서, CarNeRF는 뭘 하는가?

접근은 단순합니다.

- 딜러를 없앤다 → **거래 수수료 0%** (당근마켓처럼 저렴하게)
- 대신 AI가 딜러의 역할을 한다 → **차량 상태 검증 + 가격 진단** (엔카보다 정확하게)
- 전문 용어? AI가 쉬운 말로 바꿔준다 → **누구나 이해 가능**
- 사진 몇 장? 3D 모델로 360도 확인 → **비대면으로도 실차 확인 수준**

한 줄로 요약하면, "**당근마켓의 가격 + 엔카의 신뢰성**"을 AI 기술로 실현하는 것입니다.

## 이 시장, 얼마나 큰가?

### TAM — 전체 시장: 약 30~44조원

국내 중고차 시장은 연간 약 **240만대**가 실거래되고 있습니다 (다나와 글로벌오토뉴스, 2024). 신차 시장(약 170만대)의 1.4배 규모입니다. 금액으로 환산하면, 글로벌 시장조사기관 **Frost & Sullivan**은 2022년 기준 **44조원**으로 추정했고, **Mordor Intelligence**는 2025년 기준 **USD 24.1B(약 31조원)**으로 보고 있습니다. 보수적으로 잡아도 **30조원 이상**인 시장입니다.

> 출처:

### SOM — 우리의 초기 타겟: 20-30대 직거래 시장

온라인 중고차 시장(~5조원) 안에서 저희가 처음 공략할 대상은 **20-30대 첫 차 구매자**입니다.

왜 이 세대인가?

- 중고차 앱 '첫차' 데이터에 따르면, 20-30대 수도권 소비자의 중고차 평균 구매가는 **1,447만원** ([프레스맨, 2024](http://www.pressm.kr/news/articleView.html?idxno=12187))
- KB차차차 기준 20대 중고차 조회 1위는 **아반떼**, 30대부터는 **그랜저** ([디지털타임스, 2024](https://www.dt.co.kr/article/11592108))
- MZ세대 5명 중 1명이 500~1,000만원 사이 중저가 중고차를 구매 — 가격에 민감하면서도 온라인 구매에 익숙한 세대

이 세대는 "차를 사고 싶은데 뭘 봐야 하는지 모르겠다"는 니즈가 가장 강합니다. 전문 지식은 부족하지만 디지털 서비스에는 익숙하고, 딜러보다 직거래를 선호합니다. **저희 서비스가 정확히 해결해주는 문제를 가진 세대**입니다.

온라인 중고차 시장 5조원 중 20-30대 비중을 보수적으로 30%로 잡으면 약 **1.5조원**, 그중 직거래 비중을 고려하면 수천억원 규모의 시장이 저희의 초기 무대입니다. 여기서 시작해 소규모 딜러 → 중대형 딜러 순으로 확장하는 것이 로드맵입니다.

# 2. 실현가능성

## 구체적으로 어떻게 가능한가?

"AI가 딜러를 대신한다"고 했는데, 이걸 실제로 어떻게 구현했는지 설명드리겠습니다.

**1) 3D 차량 뷰어 — 사진이 아니라 3D로 본다**

판매자가 스마트폰으로 차량을 한 바퀴 촬영하면, 저희 시스템이 그 영상에서 자동으로 고품질 3D 모델을 만들어냅니다. 구매자는 웹에서 차량을 360도 자유롭게 돌려보며 확인할 수 있습니다. 사진 5장 보고 판단하는 것과는 차원이 다른 경험입니다.

기술적으로는 COLMAP으로 카메라 위치를 추정하고, 3D Gaussian Splatting으로 포토리얼리스틱한 3D 모델을 생성합니다.

**2) AI 결함 탐지 — 눈으로 놓치는 것도 AI는 잡는다**

3D 모델을 여러 각도에서 렌더링한 뒤, YOLOv8 기반 AI가 스크래치, 덴트, 도색 변색 등을 자동으로 감지합니다. 감지된 결함은 3D 뷰어 위에 마커로 표시되어, "여기에 이런 흠집이 있구나"를 직관적으로 확인할 수 있습니다.

**3) AI 가격 진단 — "이 가격이 맞는 건가?"에 대한 답**

동일 모델의 과거 거래 기록, 연식, 주행거리, 결함 상태를 종합 분석하여 적정 가격을 산출합니다. "이 차는 시세 대비 200만원 비쌉니다" 같은 판단을 AI가 내려줍니다. 직거래의 가장 큰 불신 — 가격에 대한 불확실성 — 을 해소하는 기능입니다.

**4) AI 차량 요약 — 전문 용어 없이, 쉬운 말로**

ChatGPT API를 활용해 해당 차량의 장단점, 알려진 고질병, 실사용자 후기를 종합하여 누구나 이해할 수 있는 말로 요약해줍니다. 성능기록부를 해독할 필요가 없어집니다.

```
판매자 스마트폰 촬영
    │
    ▼
① 영상 → 프레임 추출 (2fps)
② COLMAP Structure-from-Motion
③ 3D Gaussian Splatting 학습
④ AI 결함 탐지 (YOLOv8)
⑤ AI 가격 진단 (LightGBM)
⑥ ChatGPT 차량 분석 요약
    │
    ▼
구매자: 웹/앱에서 3D 뷰어 + AI 진단 결과 확인
```

그리고 이건 구상만 한 게 아닙니다. **실제로 동작하는 프로토타입을 이미 완성했습니다.**

| 항목 | 상태 |
|------|------|
| 3D Gaussian Splatting 파이프라인 | 완료 (표준 45분, HQ 140분) |
| 웹 3D 뷰어 + 결함 마커 | 완료 |
| AI 결함탐지 / 가격예측 / 차량요약 API | 완료 |
| 포인트 충전/사용 + 후기 시스템 | 완료 |
| 풀스택 웹 서비스 (FastAPI + Tailwind CSS) | 완료 |

| 분류 | 기술 |
|------|------|
| 3D 파이프라인 | PyTorch, CUDA 12.2, pycolmap, 3D Gaussian Splatting |
| AI 모델 | YOLOv8, SAM, LightGBM, scikit-learn |
| AI 분석 | OpenAI ChatGPT API (gpt-4o-mini) |
| Backend | FastAPI, SQLAlchemy, Python 3.11 |
| Frontend | Jinja2 SSR, Tailwind CSS, Three.js |
| 인프라 | NVIDIA A100 80GB x2 |

## 그러면 기존 서비스랑 뭐가 다른가?

| | 엔카 | KB차차차 | 당근마켓 | 헤이딜러 | **CarNeRF** |
|------|------|---------|---------|---------|------------|
| 차량 확인 | 사진 | 사진 | 사진 | 사진 | **3D 모델** |
| 결함 확인 | 성능기록부 (전문용어) | 진단서 | 없음 | 딜러 검수 | **AI 자동탐지** |
| 가격 투명성 | 시세표 | 시세표 | 없음 | 딜러 견적 | **AI 시세분석** |
| 거래 수수료 | 등록비/광고비 | 무료 | 무료 | 딜러 수수료 | **0%** |
| 정보 접근성 | 전문가용 | 전문가용 | 판매자 주관 | 딜러 설명 | **AI가 쉽게 요약** |

"3D + AI 진단 + 수수료 무료"를 동시에 제공하는 서비스는 국내에 없습니다. 이 조합 자체가 저희의 진입장벽이자 차별화 포인트입니다.

# 3. 성장전략

## 돈은 어떻게 버는가?

거래 자체에는 수수료를 받지 않습니다. 이것이 사용자를 끌어오는 핵심입니다.

수익은 **AI 부가서비스**에서 만듭니다. 사용자가 포인트를 충전하고(1,000원 = 1,000P), AI 서비스를 이용할 때 차감하는 구조입니다.

| AI 서비스 | 포인트 | 사용자 입장의 가치 |
|-----------|--------|-------------------|
| AI 차량 분석 | 500P | "이 차 살 만한가?" 장단점 요약 |
| AI 가격 진단 | 300P | "이 가격이 적정한가?" 시세 판단 |
| AI 파손 진단 | 1,000P | "어디가 망가졌나?" 3D 결함 + 수리비 예측 |
| 프리미엄 상단 배치 | 5,000P | 판매자: 7일간 검색 최상단 노출 |

중고차는 수백만원~수천만원짜리 거래입니다. 그 거래의 안심을 위해 500원~1,000원을 쓰는 건 사용자 입장에서 매우 합리적인 비용입니다. "1,000원으로 파손 진단을 받아서 숨겨진 사고를 발견했다"면, 그 1,000원은 수백만원의 가치를 합니다.

서비스가 성장하면 수익원도 자연스럽게 확장됩니다:

| 수익원 | 시점 | 월 매출 목표 |
|--------|------|-------------|
| AI 서비스 수수료 | 2026.09~ | 200만원 |
| 판매자 프리미엄 구독 (월 29,900원) | 2027.01~ | 500만원 |
| 딜러 구독 (월 99,000원) | 2027.03~ | 300만원 |
| 거래 성사 수수료 (0.5~1%) | 2027.06~ | 1,000만원 |
| 보험/금융 제휴 | 2027.06~ | 500만원 |

## 앞으로의 계획

지금 프로토타입은 완성된 상태입니다. 여기서부터 어떻게 실제 서비스로 만들어갈 것인지, 구체적인 일정입니다.

| 기간 | 뭘 하는가 | 이걸 왜 하는가 | 달성 기준 |
|------|----------|---------------|----------|
| **3/24 ~ 4/6** | 프로토타입 안정화, 발표자료 완성 | 데모 시연이 돼야 다음 단계 진행 가능 | 데모 시연 성공 |
| **4/7 ~ 4/30** | AI 모델 고도화 — 결함탐지 데이터 500장+ 수집, YOLOv8 파인튜닝, 가격예측 실데이터 학습 | 데모용 AI가 아니라 실제로 쓸 수 있는 AI로 만드는 단계 | 결함탐지 mAP 0.7+, 가격예측 R² 0.9+ |

4명이 각자 전문 영역을 맡되, 결과물은 하나로 모입니다:

```
        ┌──────────────┐
        │  PM / 기획    │  ← 요구사항, 일정, 우선순위
        └──────┬───────┘
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
┌────────┐ ┌────────┐ ┌────────┐
│3D 파이프│ │AI 개발  │ │웹 플랫폼│
│  라인   │ │        │ │        │
└───┬────┘ └───┬────┘ └───▲────┘
    │          │          │
    └──── 3D렌더/API ─────┘
```

3D 파이프라인이 모델을 만들면, AI가 그 모델에서 결함을 찾고 가격을 매기고, 웹 플랫폼이 이 모든 걸 사용자에게 보여줍니다. PM이 전체 방향과 일정을 잡습니다. 각자 독립적으로 개발하되 API 규격만 맞추면 되는 구조라서, 병렬 개발이 가능합니다.
