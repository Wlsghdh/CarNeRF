## 1. 데이터 형식 선택 (영상 vs 사진)

CarNeRF는 **영상**과 **사진** 두 가지 입력 형식을 모두 지원합니다.

### 영상 촬영

| 항목 | 내용 |
|------|------|
| **장점** | 빠른 촬영, 연속적인 시점 커버리지 확보 용이 |
| **단점** | 모션 블러 발생 가능, 불필요한 프레임 포함 |
| **권장 형식** | MP4 (H.264 코덱) |
| **권장 해상도** | 1920x1080 (Full HD) 이상 |
| **권장 프레임레이트** | 30fps 이상 (추출 시 2~3fps로 다운샘플링) |
| **촬영 시간** | 차량 1대당 1~3분 |

**영상 촬영 시 주의사항:**
- 천천히 이동하면서 촬영 (빠른 이동은 모션 블러 유발)
- 가급적 일정한 속도로 이동
- 갑작스러운 방향 전환 지양
- 촬영 시작/종료 시 잠시 정지

### 사진 촬영

| 항목 | 내용 |
|------|------|
| **장점** | 고해상도, 블러 없는 선명한 이미지 확보 가능 |
| **단점** | 촬영 시간이 더 오래 걸림, 시점 누락 가능 |
| **권장 형식** | JPEG 또는 PNG |
| **권장 해상도** | 3000x2000 이상 (DSLR/미러리스 권장) |
| **촬영 방식** | 차량 주변을 돌며 10~15도 간격으로 촬영 |

**사진 촬영 시 주의사항:**
- 동일한 카메라 설정(초점 거리, 화이트밸런스) 유지
- 자동 초점보다 수동 초점 권장
- 연속 촬영(burst) 모드 활용

### 형식별 권장 상황

| 상황 | 권장 형식 |
|------|----------|
| 빠른 촬영이 필요한 경우 | 영상 |
| 최고 품질이 필요한 경우 | 사진 |
| 스마트폰으로 촬영 | 영상 |
| DSLR/미러리스 카메라 사용 | 사진 |
| 차량 외관 전체를 빠르게 캡처 | 영상 |
| 차량의 특정 부분을 세밀하게 캡처 | 사진 |

## 2. 촬영 가이드라인

### 2.1 촬영 각도

차량의 모든 면을 빠짐없이 촬영하는 것이 가장 중요합니다.

```
                    정면 (Front)
                        |
                   ┌────┴────┐
                   │         │
          좌측면 ──│  차량    │── 우측면
         (Left)   │  (Car)   │   (Right)
                   │         │
                   └────┬────┘
                        |
                    후면 (Rear)

    ────────────────────────────────────
    촬영 경로: 차량 주변을 360도 한 바퀴 이상 회전
    ────────────────────────────────────
```

**수평 각도 (Azimuth):**

```
        촬영 포인트 (최소 24개 위치, 15도 간격)

                    o   o   o
                o               o
            o                       o
           o                         o
          o       ┌───────────┐       o
          o       │           │       o
          o       │   차량     │       o
          o       │           │       o
          o       └───────────┘       o
           o                         o
            o                       o
                o               o
                    o   o   o
```

- **최소**: 8개 위치 (45도 간격) - 품질 낮음
- **권장**: 24개 위치 (15도 간격) - 양호
- **최적**: 36개 위치 (10도 간격) - 우수

**수직 각도 (Elevation):**

3가지 높이에서 촬영하는 것을 권장합니다.

```
    높은 각도 (30~45도)  ───  o  ─── 지붕/후드 디테일 확보
                              \
                               \
    중간 각도 (10~20도)  ───  o ── 차량 측면 주요 시점
                              |
                              |
    낮은 각도 (0~5도)    ───  o  ─── 범퍼/하부 디테일 확보

                         ┌───────┐
                         │ 차량   │
                         └───────┘
```

| 수직 각도 | 촬영 대상 | 필수 여부 |
|-----------|----------|----------|
| 낮은 각도 (0~5도) | 하부, 범퍼, 휠, 타이어 | 선택 |
| 중간 각도 (10~20도) | 차체 측면, 도어, 창문 | **필수** |
| 높은 각도 (30~45도) | 지붕, 후드, 트렁크 | 권장 |

### 2.2 조명 조건

조명은 3D 재구성 품질에 매우 큰 영향을 미칩니다.

**권장 조건:**

| 조건 | 권장도 | 설명 |
|------|--------|------|
| 흐린 날 야외 | 최고 | 균일한 조명, 그림자 최소화 |
| 그늘진 주차장 | 좋음 | 균일한 간접 조명 |
| 맑은 날 아침/저녁 | 보통 | 부드러운 자연광 (golden hour) |
| 맑은 날 한낮 | 나쁨 | 강한 그림자, 반사 과다 |
| 야간/실내 형광등 | 나쁨 | 불균일한 조명, 색온도 문제 |
| 직사광선 + 그림자 혼재 | 최악 | 극심한 명암 차이 |

**조명 관련 핵심 원칙:**

1. **균일한 조명**: 차량 전체가 비슷한 밝기로 조명되어야 합니다.
2. **강한 반사 방지**: 차량 도장면의 정반사(specular reflection)가 과도하면 재구성 품질이 저하됩니다.
3. **일관된 조명**: 촬영 중 조명 조건이 변하면 안 됩니다 (구름이 지나가는 동안은 촬영 중단).

### 2.3 촬영 거리

```
    너무 가까움 (< 1m)        적정 거리 (2~4m)         너무 멀음 (> 8m)

    ┌──────────┐             ┌──────────┐             ┌──────────┐
    │  ┌────┐  │             │          │             │          │
    │  │부분│  │             │ ┌──────┐ │             │  ┌──┐    │
    │  │만  │  │             │ │ 전체  │ │             │  │  │    │
    │  │보임│  │             │ │ 차량  │ │             │  └──┘    │
    │  └────┘  │             │ └──────┘ │             │ 작게 보임 │
    │          │             │          │             │          │
    └──────────┘             └──────────┘             └──────────┘

    X 부분적 커버리지         O 이상적인 커버리지        X 디테일 부족
    X SfM 실패 가능           O SfM 안정적               X 해상도 낭비
```

| 촬영 거리 | 평가 | 비고 |
|-----------|------|------|
| 1m 미만 | 부적합 | 차량 일부만 보여 SfM 매칭 실패 가능 |
| 1~2m | 보통 | 근접 디테일 보충 촬영 시 적합 |
| **2~4m** | **최적** | **차량 전체가 프레임에 들어오면서 디테일 확보** |
| 4~6m | 보통 | 전체 맥락 파악용 보충 촬영 시 적합 |
| 6m 초과 | 부적합 | 차량 디테일 해상도 부족 |

### 2.4 시점 커버리지 (Overlap)

인접한 이미지 간 **60~80% 이상 겹침(overlap)**이 필수적입니다.
이는 COLMAP SfM이 이미지 간 특징점을 안정적으로 매칭하기 위해 필요합니다.

```
    좋은 커버리지 (70% 겹침)           나쁜 커버리지 (20% 겹침)

    ┌───────────┐                     ┌──────┐     ┌──────┐
    │  이미지 1  │                     │ img1 │     │ img2 │
    │     ┌──────┼──┐                 │      │     │      │
    │     │ 겹침  │  │                 └──────┘     └──────┘
    └─────┼──────┘  │
          │ 이미지 2 │                  ← 겹치는 영역 없음!
          └─────────┘                    COLMAP 매칭 실패 위험
```

### 2.5 촬영 시 피해야 할 것

- **움직이는 물체**: 행인, 다른 차량 등이 프레임에 포함되면 3D 재구성에 노이즈 발생
- **진동/흔들림**: 삼각대 사용 권장, 손 떨림 보정(OIS) 기능이 있는 카메라 활용
- **줌 변경**: 촬영 중 줌(초점 거리)을 변경하면 카메라 모델 추정이 어려워짐
- **자동 노출/화이트밸런스**: 이미지 간 색상/밝기 차이 유발, 수동 모드 권장
- **반복 패턴 배경**: 타일 바닥, 벽돌벽 등은 SfM 매칭 혼동 유발 가능

## 3. 이미지 수량 기준

### 최소/권장/최적 수량

| 기준 | 이미지 수 | COLMAP 성공률 | 재구성 품질 | 소요 시간 (A100) |
|------|----------|-------------|-----------|----------------|
| **최소** | 50장 | ~60% | 낮음 | ~15분 |
| **권장** | 150~200장 | ~90% | 양호 | ~30분 |
| **최적** | 250~350장 | ~95% | 우수 | ~45분 |
| 과다 | 500장 이상 | ~95% | 우수 (정체) | 60분+ |

> **참고:** 이미지 수가 500장을 넘으면 품질 향상은 미미하고 처리 시간만 크게 증가합니다.

### 촬영 구성 권장안

**기본 촬영 (약 150장):**

| 촬영 종류 | 수량 | 각도 | 설명 |
|-----------|------|------|------|
| 중간 높이 360도 | 72장 | 수평 5도 간격 | 차체 측면 주요 시점 |
| 높은 높이 360도 | 36장 | 수평 10도 간격 | 지붕/후드 커버리지 |
| 낮은 높이 360도 | 36장 | 수평 10도 간격 | 하부/휠 디테일 |
| 전면/후면 보충 | 6장 | 다양한 높이 | 정면/후면 디테일 |

**세밀 촬영 (약 300장):**

위 기본 촬영에 추가로:

| 촬영 종류 | 수량 | 설명 |
|-----------|------|------|
| 헤드라이트/테일라이트 근접 | 10~15장 | 복잡한 형상 디테일 |
| 엠블럼/배지 근접 | 5~10장 | 작은 디테일 |
| 휠/타이어 근접 | 15~20장 | 휠 스포크 디테일 |
| 도어 핸들/미러 근접 | 10~15장 | 돌출 부위 디테일 |
| 추가 45도 앵글 보충 | 50~80장 | 시점 커버리지 보강 |

## 4. 좋은 데이터 vs 나쁜 데이터

### 좋은 데이터 예시

```
[좋은 예시 1] 균일한 조명의 야외 촬영
 - 흐린 날씨, 차량 전체에 균일한 밝기
 - 그림자 없음, 반사 최소화
 - 차량이 프레임의 60~70%를 차지
 - 배경이 적절한 텍스처를 가짐 (주차장 아스팔트 등)
 - COLMAP 성공률: 높음

[좋은 예시 2] 체계적인 촬영 경로
 - 차량 주변 360도를 일정 간격으로 촬영
 - 3가지 높이에서 반복 촬영
 - 인접 사진 간 70% 이상 겹침
 - 모든 사진에서 초점이 정확히 맞음
 - 일관된 노출/화이트밸런스
 - COLMAP 성공률: 매우 높음

[좋은 예시 3] 디테일 보충 촬영 포함
 - 전체 촬영 (200장) + 근접 촬영 (50장)
 - 복잡한 부위(그릴, 헤드라이트, 엠블럼)에 추가 촬영
 - 근접 촬영도 주변 영역과 충분한 겹침 확보
 - 재구성 품질: 우수
```

### 나쁜 데이터 예시

```
[나쁜 예시 1] 강한 직사광선 촬영
 - 한낮 직사광선으로 차량 한쪽에 강한 그림자
 - 도장면에 태양 반사가 과도하게 보임 (specular highlight)
 - 그늘 쪽과 양지 쪽의 밝기 차이가 극심
 - 결과: 그림자/반사가 3D 형상에 잘못 베이킹됨

[나쁜 예시 2] 불충분한 시점 커버리지
 - 차량 전면과 운전석 쪽만 촬영 (약 90도 범위)
 - 후면과 조수석 쪽 데이터 없음
 - 이미지 수: 20장
 - 결과: 촬영되지 않은 영역이 빈 공간으로 남음, COLMAP 실패 가능

[나쁜 예시 3] 모션 블러가 심한 영상
 - 빠르게 걸으면서 촬영한 영상
 - 대부분의 프레임에 모션 블러 존재
 - 카메라 흔들림이 심함
 - 결과: 특징점 추출 실패, COLMAP 매칭률 급감

[나쁜 예시 4] 배경에 움직이는 물체 다수
 - 교통량이 많은 도로변에서 촬영
 - 행인, 다른 차량이 계속 프레임에 출현
 - 결과: 움직이는 물체가 3D 공간에 노이즈로 나타남

[나쁜 예시 5] 줌 변경 및 설정 변경
 - 촬영 중간에 줌인/줌아웃 수행
 - 자동 노출로 인해 이미지마다 밝기가 다름
 - 결과: 카메라 intrinsics 추정 불안정, 색상 불일치
```

## 5. 공개 테스트 데이터셋

파이프라인 테스트 및 벤치마킹에 사용할 수 있는 공개 데이터셋 목록입니다.

### 5.1 Tanks and Temples

대규모 실외/실내 장면에 대한 멀티뷰 이미지 및 Ground Truth를 제공하는 벤치마크 데이터셋입니다.

| 항목 | 내용 |
|------|------|
| **웹사이트** | [https://www.tanksandtemples.org/](https://www.tanksandtemples.org/) |
| **데이터 종류** | 영상 프레임, 카메라 파라미터, Ground Truth 메쉬 |
| **장면 예시** | Truck, Train, M60 (탱크) 등 차량 포함 |
| **용도** | 3D 재구성 품질 벤치마킹 |
| **라이선스** | 연구 목적 무료 사용 |

**다운로드:**
```bash
# Tanks and Temples 데이터 다운로드 (예: Truck 장면)
```

### 5.2 KITTI Dataset

자율주행 분야의 대표적인 데이터셋으로, 차량 관련 다양한 센서 데이터를 제공합니다.

| 항목 | 내용 |
|------|------|
| **웹사이트** | [https://www.cvlibs.net/datasets/kitti/](https://www.cvlibs.net/datasets/kitti/) |
| **데이터 종류** | 스테레오 이미지, LiDAR 포인트 클라우드, GPS/IMU |
| **장면 특징** | 도심/교외 주행 장면, 다수의 차량 포함 |
| **용도** | 차량 검출, 3D 재구성, 자율주행 벤치마킹 |
| **라이선스** | Creative Commons Attribution-NonCommercial-ShareAlike 3.0 |

**다운로드:**
```bash
# KITTI Raw Data 다운로드 (카메라 이미지)
# 공식 웹사이트에서 회원가입 후 다운로드
wget https://s3.eu-central-1.amazonaws.com/avg-kitti/raw_data/<sequence>.zip
unzip <sequence>.zip -d data/raw/kitti/
```

### 5.3 Mip-NeRF 360 Dataset

NeRF/Gaussian Splatting 연구에서 가장 널리 사용되는 벤치마크 데이터셋입니다.

| 항목 | 내용 |
|------|------|
| **웹사이트** | [https://jonbarron.info/mipnerf360/](https://jonbarron.info/mipnerf360/) |
| **데이터 종류** | 360도 촬영된 다중 시점 이미지 + COLMAP 결과 |
| **장면 예시** | bicycle, garden, stump, room, counter, kitchen, bonsai |
| **용도** | Gaussian Splatting 학습 테스트, 품질 비교 |
| **라이선스** | 연구 목적 무료 사용 |

**다운로드:**
```bash
# Mip-NeRF 360 Dataset 다운로드
wget http://storage.googleapis.com/gresearch/refraw360/360_v2.zip
unzip 360_v2.zip -d data/raw/mipnerf360/
```

> **참고:** 이 데이터셋은 이미 COLMAP 결과가 포함되어 있으므로, Step 2를 건너뛰고
> 바로 Gaussian Splatting 학습을 시작할 수 있습니다.

### 5.5 CO3D (Common Objects in 3D)

Meta Research에서 공개한 대규모 멀티뷰 데이터셋으로, 다양한 물체 카테고리(자동차 포함)를 제공합니다.

| 항목 | 내용 |
|------|------|
| **웹사이트** | [https://github.com/facebookresearch/co3d](https://github.com/facebookresearch/co3d) |
| **데이터 종류** | 영상에서 추출된 다중 시점 이미지 + 카메라 파라미터 |
| **카테고리** | 51개 카테고리 (car, truck, bus 등 차량 카테고리 포함) |
| **용도** | 물체 단위 3D 재구성 |
| **라이선스** | Apache 2.0 |

**다운로드:**
```bash
# CO3D 데이터셋 (car 카테고리만)
pip install co3d
python -c "from co3d.dataset import download; download('car', 'data/raw/co3d/')"
```

### 데이터셋 요약 비교

| 데이터셋 | 차량 포함 | COLMAP 결과 포함 | 난이도 | 추천 용도 |
|----------|---------|----------------|--------|----------|
| Tanks and Temples | O (Truck 등) | X | 중 | 벤치마킹 |
| KITTI | O (다수) | X | 상 | 자율주행 장면 |
| Mip-NeRF 360 | X (일반 장면) | **O** | 하 | **파이프라인 테스트 (권장)** |
| DTU | X (실험실 물체) | O | 하 | 정량 평가 |
| CO3D | **O (car 카테고리)** | O | 중 | **차량 3D 재구성 (권장)** |

> **파이프라인을 처음 테스트할 때는** Mip-NeRF 360 데이터셋의 `bicycle` 또는 `garden` 장면을 권장합니다.
> COLMAP 결과가 이미 포함되어 있어 빠르게 Gaussian Splatting 학습을 검증할 수 있습니다.
>
> **차량 특화 데이터로 테스트할 때는** CO3D의 `car` 카테고리를 권장합니다.
